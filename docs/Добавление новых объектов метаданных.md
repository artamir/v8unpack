# Добавление поддержки нового типа объекта метаданных

Этот документ описывает полный цикл добавления в v8unpack поддержки нового типа объекта метаданных 1С — от появления ошибки до успешного прохождения тестов.

---

## Признак того, что тип не поддержан

При разборке конфигурации инструмент выводит ошибку вида:

```
неизвестный тип вложенных метаданных: 13134204-f60b-11d5-a3c7-0050bae0a776
в файле 79b41fb6-cf3d-433c-83f2-a97b226178c4
```

Это означает, что встреченный UUID ещё не зарегистрирован в `MetaDataTypes`.

---

## Шаг 1. Подготовить тестовую конфигурацию

1. Откройте конфигурацию из каталога `tests/` в Конфигураторе 1С.
2. Добавьте нужный объект. Имя объекта должно отражать тип и вариант, например:
   - `РеквизитДокументаСтрока`
   - `ФормаРегистраСведений`
3. Добавляйте объектов ровно столько, сколько существует значимых вариантов — как правило, 2–3 экземпляра.
4. Сохраните конфигурацию и выгрузите CF.

---

## Шаг 2. Определить UUID нового типа

Запустите разборку тестового CF с параметром `temp`, чтобы сохранить промежуточные файлы:

```python
import v8unpack

v8unpack.extract('tests/test.cf', '_unpack', temp='_temp')
```

Из текста ошибки возьмите UUID неизвестного типа, например:
```
13134204-f60b-11d5-a3c7-0050bae0a776
```

Откройте файл из строки «в файле ...» — он находится в `_temp/decode_stage_2/`.
В начале файла будет имя типа метаданных на русском, например `ФормаЗаписи`.

По этому имени найдите в Синтаксис-помощнике 1С строку вида:
```
ОбъектМетаданных: РегистрСведений.Формы
```

Английское имя там будет `MetadataObject:InformationRegister.Forms` — то есть класс называется `InformationRegisterForm` (число → единственное).

---

## Шаг 3. Зарегистрировать тип в `metadata_types.py`

Откройте `src/v8unpack/metadata_types.py` и добавьте запись в `MetaDataTypes` в алфавитном порядке:

```python
class MetaDataTypes(Enum):
    ...
    InformationRegister = '13134201-f60b-11d5-a3c7-0050bae0a776'
    InformationRegisterCommand = 'b44ba719-945c-445c-8aab-1088fa4df16e'
    InformationRegisterForm = '13134204-f60b-11d5-a3c7-0050bae0a776'  # Form регистра сведений  <-- добавлено
    ...
```

Комментарий справа — обязательный, он помогает ориентироваться без поиска по синтаксис-помощнику.

---

## Шаг 4. Создать класс MetaObject

Для каждого типа метаданных в папке `src/v8unpack/MetaObject/` создаётся одноимённый файл с классом.

### 4.1 Простой тип (реквизит, атрибут и т.п.)

Если объект не хранит отдельный файл с содержимым и не имеет вложенных объектов, достаточно унаследоваться от `MetaObject` без переопределений. Скопируйте ближайший по смыслу класс:

```python
# src/v8unpack/MetaObject/InformationRegisterForm.py
from . import MetaObject


class InformationRegisterForm(MetaObject):
    pass
```

### 4.2 Сложный тип (форма, модуль, макет)

Если объект хранит отдельный бинарный или текстовый файл, необходимо переопределить методы `decode` и `encode`. За образец берите ближайший существующий класс той же природы (например, `CatalogForm` для форм или `CommonModule` для модулей).

Минимальный шаблон для формы:

```python
# src/v8unpack/MetaObject/InformationRegisterForm.py
from . import MetaObject
from .. import helper


class InformationRegisterForm(MetaObject):

    def decode(self, src_dir, dest_dir, *, version=None, **kwargs):
        # прочитать заголовок
        header_data = helper.brace_file_read(src_dir, self.container_uuid)
        self.decode_header(header_data)
        # обработать содержимое формы
        ...

    def encode(self, src_dir, dest_dir, **kwargs):
        self.encode_header()
        ...
```

### 4.3 Зарегистрировать класс в `__init__.py`

Откройте `src/v8unpack/MetaObject/__init__.py` и убедитесь, что новый класс доступен через импорт из пакета, если это требуется по соглашению проекта.

## Шаг 5. Написать тест

Создайте тест по аналогии с существующими в `tests/`. Тест должен проверять:
1. Разборку CF с новым объектом проходит без ошибок.
2. Сборка из разобранных файлов производит валидный CF.
3. Значимые поля объекта (имя, тип, реквизиты) сохраняются без потерь.

Минимальный шаблон:

```python
# tests/test_information_register_form.py
import v8unpack
from pathlib import Path

SRC_CF = Path('tests/test.cf')
UNPACK_DIR = Path('_test_unpack')
REPACK_CF = Path('_test_repack.cf')


def test_round_trip():
    v8unpack.extract(str(SRC_CF), str(UNPACK_DIR))
    v8unpack.build(str(UNPACK_DIR), str(REPACK_CF))
    assert REPACK_CF.exists()
    assert REPACK_CF.stat().st_size > 0
```

---

## Шаг 6. Запустить тесты

```bash
pytest tests/ -v
```

Основной тест разборки-сборки должен пройти без ошибок (`test_decode`).
Новый тест конкретного объекта также должен быть зелёным.

---

## Шаг 7. Обновить версию и историю

1. Поднять patch-версию в `src/v8unpack/version.py`:
   ```python
   __version__ = '1.2.9'  # было 1.2.8
   ```

2. Добавить запись в `docs/history.md`:
   ```
   1.2.9 feat поддержка типа InformationRegisterForm
   ```

   Запись обязательна — без неё PR не принимается.

## Быстрая памятка

| Действие | Файл |
|---|---|
| Регистрация UUID | `src/v8unpack/metadata_types.py` |
| Реализация сериализации | `src/v8unpack/MetaObject/<ТипОбъекта>.py` |
| Тест объекта | `tests/test_<тип>.py` |
| Версия | `src/v8unpack/version.py` |
| История | `docs/history.md` |
