# Добавление нового объекта в исходные файлы конфигурации

Этот документ описывает, как добавить новый объект метаданных (Справочник, Документ, Перечисление и т.д.) в распакованные v8unpack исходные файлы конфигурации 1С.

---

## Структура исходных файлов

После распаковки `v8unpack.extract()` конфигурация хранится как папка с подпапками по типам объектов:

```
src/
├── Configuration.json          # заголовок конфигурации + оглавление
├── Configuration.obj.bsl       # модуль сеанса
├── Configuration.app.bsl       # модуль обычного приложения
├── Catalog/
│   └── Контрагенты/
│       ├── Catalog.id.json     # uuid объекта
│       ├── Catalog.json        # заголовок + реквизиты + настройки
│       ├── Catalog.obj.bsl     # модуль объекта
│       ├── Catalog.mgr.bsl     # модуль менеджера
│       ├── Предустановленные данные.bin
│       └── CatalogForm/
│           └── ФормаЭлемента/
│               ├── CatalogForm.id.json
│               ├── CatalogForm.json
│               ├── CatalogForm.obj.bsl  # модуль формы
│               └── CatalogForm.elem.json
├── Enum/
│   └── TypeOfPrint/
│       ├── Enum.id.json
│       └── Enum.json
├── Document/
├── CommonModule/
│   └── ОбщийМодуль1/
│       ├── CommonModule.id.json
│       ├── CommonModule.json
│       └── CommonModule.obj.bsl
└── ...
```

Ключевой момент: `Configuration.json` содержит **оглавление** — явные списки UUID всех объектов каждого типа внутри `header`. Без специального режима сборщик не трогает это оглавление.

---

## Два подхода

### Подход 1 — через Конфигуратор (рекомендуется)

Это единственный безопасный способ, когда объект имеет сложную структуру (реквизиты, формы, табличные части, команды).

1. Откройте конфигурацию в 1С:Предприятие → Конфигуратор.
2. Добавьте нужный объект со всеми реквизитами и формами.
3. Сохраните конфигурацию.
4. Выгрузите CF-файл: **Конфигурация → Сохранить конфигурацию в файл**.
5. Распакуйте полученный CF:

```python
import v8unpack

v8unpack.extract('my_config.cf', 'src/')
```

6. Теперь в папке `src/` появится правильная структура нового объекта.
7. Редактируйте модули (`.bsl`) вручную, затем соберите обратно:

```python
v8unpack.build('src/', 'my_config_patched.cf')
```

### Подход 2 — напрямую через файловую систему (с `auto_include`)

Подходит для простых объектов: Общий модуль, Перечисление, Константа, простой Справочник.

При передаче `options={'auto_include': True}` при сборке v8unpack динамически строит оглавление по папкам — редактировать `Configuration.json` вручную не требуется.

> ⚠️ Ограничение: при `auto_include` вложенные объекты (формы, команды) всегда сортируются по алфавиту, порядок из конфигуратора не сохраняется.

```python
import v8unpack

# Распаковка с auto_include (оглавление не сохраняется — строится из папок)
v8unpack.extract('base.cf', 'src/', options={'auto_include': True})

# ... добавляете папку нового объекта вручную ...

# Сборка — оглавление строится автоматически по папкам
v8unpack.build('src/', 'patched.cf', options={'auto_include': True})
```

## Шаблоны файлов для ручного добавления

### Общий модуль (`CommonModule`)

Создайте папку `src/CommonModule/<ИмяМодуля>/` и три файла:

**`CommonModule.id.json`** — генерируйте UUID любым способом, например `python -c "import uuid; print(uuid.uuid4())"`:
```json
{
  "uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
}
```

**`CommonModule.json`** — скопируйте из существующего модуля и исправьте `name`, `name2`, `uuid` в `header`:
```json
{
  "name": "МойОбщийМодуль",
  "name2": {"ru": "Мой общий модуль"},
  "comment": "",
  "header": [ ... ],
  "obj_version": "803"
}
```

**`CommonModule.obj.bsl`** — текст модуля в кодировке UTF-8:
```bsl
Функция МояФункция() Экспорт
    Возврат "OK";
КонецФункции
```

### Перечисление (`Enum`)

Папка `src/Enum/<ИмяПеречисления>/`, два файла:

**`Enum.id.json`**:
```json
{
  "uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
}
```

**`Enum.json`** — скопируйте из существующего Enum того же проекта и замените `name`, UUID в `header`.

> Значения перечисления (виды) хранятся внутри `header` как вложенные записи — редактируйте их по образцу существующего файла.

### Справочник (`Catalog`) — минимальная структура

Папка `src/Catalog/<ИмяСправочника>/`, минимум два файла:

```
Catalog/МойСправочник/
├── Catalog.id.json
└── Catalog.json
```

При наличии модуля объекта добавьте:
- `Catalog.obj.bsl` — модуль объекта
- `Catalog.mgr.bsl` — модуль менеджера

Для форм — подпапку `CatalogForm/<ИмяФормы>/` со своими `CatalogForm.id.json`, `CatalogForm.json`, `CatalogForm.obj.bsl`, `CatalogForm.elem.json`.

> Структуру `CatalogForm.elem.json` (макет формы) создавайте только из конфигуратора — формат слишком сложен для ручного написания.

## Получить шаблон из существующего объекта

Проще всего получить правильный шаблон — скопировать похожий объект и переименовать:

```python
import shutil, json, uuid

# копируем существующий справочник как основу
shutil.copytree('src/Catalog/СуществующийСправочник', 'src/Catalog/НовыйСправочник')

# заменяем UUID
new_uuid = str(uuid.uuid4())
with open('src/Catalog/НовыйСправочник/Catalog.id.json', 'w', encoding='utf-8') as f:
    json.dump({'uuid': new_uuid}, f, ensure_ascii=False, indent=2)

# правим имя в Catalog.json
data = json.load(open('src/Catalog/НовыйСправочник/Catalog.json', encoding='utf-8'))
data['name'] = 'НовыйСправочник'
data['name2'] = {'ru': 'Новый справочник'}
# UUID в header нужно заменить вручную — несколько вхождений внутри data['header']
with open('src/Catalog/НовыйСправочник/Catalog.json', 'w', encoding='utf-8') as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
```

> После копирования обязательно замените все вхождения старого UUID на новый внутри `Catalog.json` (поиском по тексту файла).

## Обновление оглавления в `Configuration.json` (без `auto_include`)

Если `auto_include` не используется, после добавления объекта вручную нужно зарегистрировать его UUID в оглавлении конфигурации.

Оглавление находится в `Configuration.json` → `header[0]`. Структура каждой группы:
```
header[0] = [
  ...,
  count_groups,
  group_1,   # General
  group_2,   # Main
  ...
]
```

Каждая группа содержит список типов, каждый тип — список UUID объектов. Для добавления объекта:

1. Найдите нужный UUID типа из `metadata_types.py` (например `Catalog = "cf4abea6-37b2-11d4-940f-008048da11f9"`).
2. В блоке этого типа увеличьте счётчик и добавьте UUID нового объекта в список.

**Рекомендация:** не редактируйте `Configuration.json` вручную без крайней необходимости. Используйте либо Конфигуратор, либо `auto_include`.

## Итоговый рабочий процесс

| Задача | Рекомендуемый способ |
|---|---|
| Сложный объект с формами/реквизитами | Конфигуратор → CF → `extract()` |
| Простой модуль / изменение BSL | Напрямую в `.bsl` файлах, затем `build()` |
| Новый простой объект без форм | `auto_include` + шаблон из существующего |
| Полностью новая конфигурация | Конфигуратор → CF → `extract()` как база |
