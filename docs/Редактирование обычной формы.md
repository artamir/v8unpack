# Редактирование обычной формы объекта

Этот документ описывает, как работать с файлами обычной формы (документа, справочника и т.д.) в распакованных v8unpack исходниках.

---

## Файловая структура формы

После распаковки каждая форма находится в подпапке `<ТипФормы>/<ИмяФормы>/` внутри объекта:

```
Document/ЗаказПокупателя/
└── DocumentForm/
    └── ФормаДокумента/
        ├── DocumentForm.id.json    # uuid формы
        ├── DocumentForm.json       # заголовок формы + тип + версия
        ├── DocumentForm.obj.bsl    # модуль формы (BSL-код)
        └── DocumentForm.elem.json  # макет формы: реквизиты и элементы
```

---

## Назначение файлов

### `DocumentForm.id.json`

UUID формы. Не изменять вручную.

```json
{
  "uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
}
```

### `DocumentForm.json`

Метаданные формы: имя, тип формы, версия элементов.

```json
{
  "name": "ФормаДокумента",
  "name2": {"ru": "Форма документа"},
  "comment": "",
  "Тип формы": 1,
  "Версия элементов формы": 27,
  "obj_version": "13",
  "header": [ ... ]
}
```

Поля:
- `"Тип формы"` — 1 = форма документа, 2 = форма списка, 3 = форма выбора.
- `"Версия элементов формы"` — соответствует версии платформы (27 = 8.2 с обычными формами). **Не менять.**

### `DocumentForm.obj.bsl`

Модуль формы. Обычный текстовый файл UTF-8. Это единственный файл формы, который безопасно редактировать вручную:

```bsl
Процедура ПриОткрытии()
    ЛокЗначение = "TEST_FORM_01";
КонецПроцедуры

Процедура КнопкаВыполнитьНажатие(Кнопка)
    // ...
КонецПроцедуры
```

### `DocumentForm.elem.json`

Макет формы — реквизиты и визуальные элементы. Структура:

```
{
  "params":   [],                  // параметры формы (обычно пусто)
  "props":    [ ... ],             // реквизиты формы
  "commands": [ ... ],             // команды формы
  "tree":     [ ... ],             // дерево элементов (плоский список с иерархией)
  "data":     { "Страница1": ...,  // данные каждого элемента по ключу "Страница/ИмяЭлемента"
                "Страница1/Number": ...,
                ... }
}
```

---

## Структура `elem.json` подробно

### `props` — реквизиты формы

Каждый реквизит — объект с полями:

```json
{
  "name": "СтатусЗаказаРеквизитФормы",
  "id":   "80",
  "raw":  [
    ["80"],      // id
    "0",         // 0 = ссылочный тип (объект/перечисление), 1 = примитив (Булево, Строка...)
    "0",         // флаги
    "1",         // признак доступности
    "\"СтатусЗаказаРеквизитФормы\"",
    ["\"Pattern\"", ["\"#\"", "<uuid-типа>"]]  // тип реквизита
  ]
}
```

Тип реквизита:
- `["\"Pattern\"", ["\"#\"", "<uuid>"]]` — ссылка на объект метаданных (справочник, перечисление и т.д.)
- `["\"Pattern\"", ["\"B\""]]` — Булево
- `["\"Pattern\"", ["\"S\"", "50", "0"]]` — Строка(50)
- `["\"Pattern\"", ["\"N\"", "10", "2", "1"]]` — Число(10,2)

### `tree` — дерево элементов формы

Плоский список описаний элементов. Каждый элемент содержит:

```json
{
  "name":  "Покупатель",
  "type":  "Field",
  "page":  "Страница1",
  "child": [ ... ]   // для Panel, Table — вложенные элементы
}
```

Типы элементов:
- `"Field"` — поле ввода (связано с реквизитом через `"prop"` в `data`)
- `"Label"` — надпись
- `"Table"` — табличная часть
- `"Panel"` — панель со страницами
- `"Button"` — кнопка

### `data` — данные элементов

Ключ — путь `"Страница/ИмяЭлемента"`. Значение — объект с:

```json
{
  "id":   "2",          // числовой id элемента (уникален в форме)
  "ver":  27,           // версия формата
  "page": "Страница1",  // страница, на которой расположен элемент
  "raw":  [ ... ],      // бинарные данные: размеры, привязки, свойства
  "prop": "ДокументОбъект"  // только для Field: путь к реквизиту-источнику данных
}
```

## Что можно безопасно редактировать вручную

| Задача | Файл | Безопасность |
|---|---|---|
| Добавить/изменить процедуру | `DocumentForm.obj.bsl` | ✅ Безопасно |
| Переименовать процедуру | `DocumentForm.obj.bsl` | ✅ Безопасно |
| Изменить текст надписи | `DocumentForm.elem.json` → `data["Страница1/НадписьХ"]` | ⚠️ Осторожно |
| Добавить реквизит формы | `DocumentForm.elem.json` → `props` | ⚠️ Осторожно |
| Добавить/удалить элемент формы | `DocumentForm.elem.json` → `tree` + `data` | ❌ Только через конфигуратор |
| Изменить привязки, размеры | `DocumentForm.elem.json` → `data[...]["raw"]` | ❌ Только через конфигуратор |

---

## Изменение текста надписи

Найти элемент по ключу в `data`, внутри `raw` найти строковый литерал с текстом. В надписи `НадписьНомер` текст хранится в блоке `raw[2][0][2]`:

```json
[
  "1",
  "2",
  ["\"ru\"", "\"Номер заказа:\""],  // <-- текст по-русски
  ["\"ro\"", "\"CMD-2008-\""]       // текст на другом языке (если есть)
]
```

Отредактируйте только строковый литерал в кавычках, не трогая структуру массива.

---

## Добавление реквизита формы

Добавьте запись в `props`. Скопируйте существующий реквизит похожего типа и измените:
- `"name"` — уникальное имя
- `"id"` — уникальный числовой id (больше максимального существующего)
- тип в `raw[5]`

Пример булевого реквизита:

```json
{
  "name": "МойФлаг",
  "id":   "120",
  "raw":  [
    ["120"],
    "1",
    "0",
    "1",
    "\"МойФлаг\"",
    ["\"Pattern\"", ["\"B\""]]
  ]
}
```

> ⚠️ После добавления реквизита в `props` он не появится на форме автоматически. Чтобы показать его как поле — нужно добавить элемент `Field` в `tree` и запись в `data`, что безопасно делать только через конфигуратор.

## Типичный рабочий процесс

### Только правка модуля формы

1. Открыть `DocumentForm.obj.bsl` в редакторе.
2. Добавить / изменить процедуры.
3. Сохранить файл.
4. Собрать CF:
   ```python
   import v8unpack
   v8unpack.build('src/', 'patched.cf')
   ```

### Изменение макета (добавление поля на форму)

Редактировать `DocumentForm.elem.json` вручную для изменения макета нецелесообразно — структура `raw` каждого элемента содержит координаты, привязки и множество бинарных параметров, формат которых описан в [Структура обычной формы.md](Структура%20обычной%20формы.md).

**Рекомендуемый подход:**

1. Открыть конфигурацию в Конфигураторе 1С.
2. Внести изменения в форму визуально.
3. Сохранить конфигурацию, выгрузить CF.
4. Распаковать: `v8unpack.extract('new.cf', 'src/')` — получить обновлённый `elem.json`.
5. Вручную отредактировать только `DocumentForm.obj.bsl` если нужно.
6. Собрать обратно: `v8unpack.build('src/', 'patched.cf')`.

### Патч модуля без пересборки через CfPatcher

Если нужно изменить только BSL-модуль формы в уже готовом CF-файле, не трогая всё остальное:

```python
from v8unpack import CfPatcher

with CfPatcher('input.cf', 'output.cf') as p:
    uuid = p.find_module_uuid("уникальная строка из модуля формы")
    code = p.read_module(uuid)
    code += "\nПроцедура МояНоваяПроцедура()\nКонецПроцедуры\n"
    p.write_module(uuid, code)
```

`CfPatcher` работает на уровне контейнера CF без полной распаковки — изменения бинарно точны.

## Версии форм

Поле `"Версия элементов формы"` в `DocumentForm.json` и поле `"ver"` в каждом элементе `data` — это версия платформы, в которой была сохранена форма.

| `ver` | Версия платформы |
|---|---|
| 27 | 8.2 (обычные формы) |
| 26 | 8.1 |

Если `ver` отличается между элементами и заголовком — форма содержит элементы разных версий. **Не изменять `ver` вручную.**
